# 文档元数据模板

> 供 Project Incubator 项目文档和未来被孵化项目的权威文档复用

## 文档状态

| 字段 | 填写要求 |
| --- | --- |
| 所属项目 | 项目的正式名称 |
| 所有者 Phase | 负责形成和治理本文档的 Phase |
| 文档状态 | Draft、Active、Superseded 或 Archived |
| 权威范围 | 本文档唯一负责维护的事实或决定 |
| 消费 Phase | 需要读取本文档的后续 Phase；未知时写“按需” |
| 更新条件 | 哪些事件会触发本文档更新 |
| 依赖文档 | 上游权威文档的准确路径；没有时写“无” |
| 最后更新 | 使用 `YYYY-MM-DD` |

## 使用规则

- 删除模板说明文字后再形成正式项目文档；
- 不把未确认假设写成已经确认的事实；
- 同一权威范围不得由多个 Active 文档重复维护；
- 文档被替代时，将状态改为 Superseded，并指向新的权威文档；
- 文档归档时，将状态改为 Archived，不再作为当前执行输入；
- 具体文档的正文结构由对应文档类型模板或当前 Phase 规则决定。
