# Maker 任务启动 Prompt 模板

> 供 Maker 在新的独立任务中表达本次任务意图；Agent 的通用启动、状态恢复和 Git 治理仍由项目根目录 `AGENTS.md` 定义。

## 文档状态

| 字段 | 当前值 |
| --- | --- |
| 所属项目 | Project Incubator 及未来使用其 Framework 的项目 |
| 所有者 Phase | Phase 0 — Idea Capture |
| 文档状态 | Draft |
| 权威范围 | Maker 启动独立任务时使用的最小 Prompt 结构 |
| 消费 Phase | Phase 0–9 |
| 更新条件 | 真实任务验证发现字段不足、职责重复或使用边界不清 |
| 依赖文档 | 项目根目录 `AGENTS.md`、`DOCS/PROJECT_STATE.md` |
| 最后更新 | 2026-07-20 |

## 1. 目标已经明确时

复制以下内容，并填写本任务特有的信息：

```text
请按照项目根目录 AGENTS.md 启动并继续当前项目。

本任务目标：
[用一句话描述本次唯一希望达成的结果]

特别注意：
[只填写本任务特有的边界、风险或限制；没有时删除本段]

需要我确认：
[列出必须由 Maker 决定、接受或验收的事项；没有时删除本段]
```

## 2. 尚不确定下一项任务时

复制以下内容：

```text
请按照项目根目录 AGENTS.md 启动并恢复当前项目状态。

请根据当前状态向我说明下一项建议，并等待我确认任务目标后再开始写入。
```

## 3. 使用规则

- 触发时机：当 Maker 请求生成、整理或提供用于新任务、新会话、新线程、阶段交接或任务交接的 Prompt 指令时，Agent 必须先读取本模板；
- 核心原则：Prompt 只承载本次任务的 delta；项目状态由 `AGENTS.md` 与 `DOCS/PROJECT_STATE.md` 恢复；
- 默认使用“目标已经明确”入口；只有需要 Agent 协助选择下一项任务时，才使用“尚不确定下一项任务”入口；
- Maker 只填写本任务特有的信息，不重复 `AGENTS.md` 已经规定的启动顺序、状态读取、角色切换或 Git 操作；
- 不在 Prompt 中搬运项目状态、当前 Phase、AI 角色、权威文档集合、Git 历史、已完成内容或其他应由权威文档恢复的信息；
- 如果本次指令与仓库权威状态冲突，Agent 必须报告冲突并等待 Maker 决定，不得静默覆盖；
- 使用本模板只表示 Maker 发起任务，不代表 Maker 已接受最终 Diff，也不代表批准 Phase 切换；
- 模板当前为 Draft。完成真实任务验证并经 Maker 复核后，才能提升为稳定模板。
