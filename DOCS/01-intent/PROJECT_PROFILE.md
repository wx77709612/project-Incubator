## 1. 文档职责

本文档定义 Project Incubator 的主要项目类型和适用路径，供后续 Phase 选择合适的协作深度。项目动机、非目标、个人约束和成功标准由 `DOCS/01-intent/INTENT.md` 唯一维护。

## 2. 主要项目类型

Project Incubator 的主要项目类型是 **Personal Tool**：面向 Maker 自用的项目孵化与 AI 协作治理系统。

它同时积累可复用的 Framework 能力，但当前不将其定义为面向外部用户的商业产品。只有在 Maker 通过真实使用跑通、跑顺流程后，才考虑将其固化为可对外分享的 Skill。

## 3. 第一目标使用者

第一目标使用者只有 Maker 自己。

未来可能服务于遇到相似协作与项目治理困难的其他创作者，但他们不是当前版本必须服务或验证的对象。

## 4. 项目规模与适用对象

- 当前规模：个人使用、单一 Maker 驱动；
- 主场景：从一个点子开始，将其孵化为与项目类型相称的结束结果；
- 次要场景：接入已有项目，基于已有材料重新澄清未解决的目标、用户、边界或商业可行性问题，并按需补充项目文档；
- 已有项目接入的具体流程、自动阶段匹配和文档补全机制尚未设计，不作为当前版本已实现能力。

## 5. 公开与商业化程度

- 公开程度：当前为 Maker 自用；未来可考虑将经真实验证的流程固化为对外分享的 Skill；
- 商业化程度：Project Incubator 自身当前不以商业化为目标；被孵化项目是否商业化由该项目的 Maker 另行决定。

## 6. 推荐流程路径

| Phase | 当前推荐深度 | 说明 |
| --- | --- | --- |
| Phase 0 — Idea Capture | 适用 | 捕获新点子；已有项目接入的处理方式留待后续设计。 |
| Phase 1 — Intent Discovery | 适用 | 明确动机、类型、对象、边界与成功标准。 |
| Phase 2 — Explore | 轻量 | 基于 Maker 的真实工作流理解问题。 |
| Phase 3 — Validate | 可跳过或轻量 | 对 Personal Tool 不要求市场、付费或商业验证；可进行自用验证。 |
| Phase 4 — Design | 按需执行 | 具体成果需要设计时执行，不预先跳过。 |
| Phase 5 — Planning | 按需执行 | 具体成果需要规划时执行，不预先跳过。 |
| Phase 6 — Build | 按需执行 | 具体成果需要制作或实现时执行，不预先跳过。 |
| Phase 7 — Launch | 轻量 | 优先确保 Maker 能实际使用；不要求公开发布或营销。 |
| Phase 8 — Iterate | 按真实使用决定 | 至少经过一轮真实使用反馈后再决定是否进入。 |
| Phase 9 — Archive | 当前不安排 | 在项目结束、暂停或需要复盘时再决定。 |
